## 🔧 核心功能详解

### 1️⃣ 亚马逊链接生成

**输入参数：**
- ASIN (必填) - 产品唯一标识
- 产品标题 - 用于 SEO 友好的 URL
- 关键词 - 搜索来源关键词
- 推广标签 (tag) - 联盟营销标识

**输出链接结构：**
```
https://www.amazon.com/产品标题/dp/ASIN
/ref=sr_1_1?                ← 搜索排名
  crid=ABC123               ← 搜索会话ID
  &keywords=关键词          ← 搜索关键词
  &qid=1234567890          ← 时间戳
  &sprefix=关键词%2Caps     ← 搜索前缀
  &sr=1-1                   ← 排名位置
  &tag=推广标签             ← 联盟标签
  &xpid=xyz                 ← 页面ID
```

---

### 2️⃣ 链接解析功能

**可解析的参数：**

| 参数 | 含义 | 示例 |
|------|------|------|
| `ref` | 来源页面 | `sr_1_1` = 搜索第1页第1位 |
| `qid` | 搜索时间戳 | Unix 时间戳 |
| `sr` | 搜索排名 | `1-1` = 第1页第1位 |
| `crid` | 搜索会话ID | 16位随机字符 |
| `keywords` | 搜索关键词 | URL 编码的关键词 |
| `sprefix` | 搜索前缀 | 关键词+类目+位置 |
| `dib` | 设备指纹 | Base64 编码的 JSON |
| `tag` | 联盟标签 | 推广者标识 |

---

### 3️⃣ 短链生成服务

**支持的短链服务：**
- TinyURL
- is.gd
- v.gd

**自建短链服务：**
- 基于 Flask + SQLite
- 支持自定义域名
- 点击统计功能
- 完全可控

## 🔬 技术实现原理

### 链接参数生成逻辑

```python
# 1. 生成时间戳
qid = int(time.time())

# 2. 生成随机会话ID
crid = generate_random_string(16)

# 3. 生成页面ID
xpid = generate_random_string(16)

# 4. 构建链接
link = f"https://amazon.com/dp/{asin}"
link += f"/ref=sr_{rank}"
link += f"?crid={crid}"
link += f"&keywords={encoded_keyword}"
link += f"&qid={qid}"
link += f"&sprefix={sprefix}"
link += f"&sr={rank}"
link += f"&tag={tag}"
link += f"&xpid={xpid}"
```

---

### dib 参数详解

**dib = Device/Browser Identifier**

```json
{
  "v": "1",           // 版本
  "ct": "search",     // 点击类型
  "t": 1234567890,    // 时间戳
  "rk": 123456        // 排名标识
}
```

**编码方式：**
- JSON → Base64 → URL 编码
- 包含设备指纹和会话信息

## ⚠️ 风险与合规

### 亚马逊能看到的数据

| 数据类型 | 说明 | 风险等级 |
|---------|------|---------|
| 访问来源 | 搜索关键词、引荐页面 | 🟡 中 |
| 搜索时间 | qid 时间戳 | 🟢 低 |
| 点击位置 | 搜索排名 sr | 🟡 中 |
| 用户行为 | 鼠标移动、停留时间 | 🔴 高 |
| 账号信息 | 账号年龄、购买历史 | 🔴 高 |
| 设备指纹 | 浏览器、操作系统 | 🔴 高 |
| IP 地址 | 地理位置、网络类型 | 🔴 高 |

---

### 真正触发风控的因素

**🔴 高风险行为：**
- 新账号大量下单
- 秒拍秒付（不浏览直接购买）
- 同一 IP 多账号操作
- 使用机房 IP 或代理 IP
- 虚拟信用卡支付
- 收货地址雷同
- 评价内容雷同
- 好评率 100%

**🟡 中风险行为：**
- 频繁更换账号
- 反复充值礼品卡
- 频繁修改收货地址

**🟢 低风险：**
- 链接参数本身
- 正常的浏览行为

## 💡 最佳实践建议

### 1. 链接生成建议

✅ **应该做的：**
- 使用真实的关键词
- 生成合理的搜索排名
- 包含完整的时间戳和会话ID
- 使用自然的浏览路径

❌ **不应该做的：**
- 使用相同的关键词重复生成
- 时间戳过于规律
- 排名位置不合理（如第100页第1位）

---

### 2. 行为模拟建议

✅ **真实用户行为：**
- 搜索关键词
- 浏览多个产品
- 查看产品详情
- 阅读评价
- 比较价格
- 加入购物车
- 稍后购买

❌ **机器人行为：**
- 直接打开链接购买
- 秒拍秒付
- 不浏览直接下单
- 固定时间间隔操作

---

### 3. 账号安全建议

✅ **账号养护：**
- 使用老账号（6个月以上）
- 正常的购买历史
- 真实的收货地址
- 真实的支付方式
- 正常的评价习惯

❌ **危险操作：**
- 新号直接测评
- 同一设备多账号
- 同一 IP 多账号
- 虚拟信用卡

---

### 4. 网络环境建议

✅ **安全网络：**
- 家庭宽带 IP
- 手机流量 IP
- 真实的地理位置

❌ **危险网络：**
- 机房 IP
- 公共代理
- VPN（特别是免费 VPN）
- 云服务器 IP

## 📊 数据流程

### 链接生成流程

```
用户输入参数
    ↓
验证 ASIN 格式
    ↓
生成随机参数 (crid, qid, xpid)
    ↓
编码关键词
    ↓
构建完整 URL
    ↓
返回生成结果
```

```
输入亚马逊链接
    ↓
正则提取 ASIN
    ↓
解析 URL 参数
    ↓
解码 URL 编码
    ↓
分析 dib 参数
    ↓
返回结构化数据
```

## ❓ 常见问题

**Q: 生成的链接安全吗？**
A: 链接参数本身安全，但行为更重要。模拟真实用户行为是关键。

**Q: 会被亚马逊封号吗？**
A: 链接不会导致封号，但异常行为会。遵循最佳实践可降低风险。

**Q: 支持哪些亚马逊站点？**
A: 支持美国、中国、日本、英国、德国、法国、意大利、西班牙等。

**Q: 短链服务稳定吗？**
A: 自建短链服务完全可控，建议使用自己的域名。
