🔧 核心功能详解

1️⃣ 亚马逊链接生成

输入参数

  • ASIN (必填) - 产品唯一标识
  • 产品标题 - 用于 SEO 友好的 URL
  • 关键词 - 搜索来源关键词
  • 推广标签 (tag) - 联盟营销标识

输出链接结构

https://www.amazon.com/产品标题/dp/ASIN /ref=sr_1_1? ← 搜索排名 crid=ABC123 ← 搜索会话ID &keywords=关键词 ← 搜索关键词 &qid=1234567890 ← 时间戳 &sprefix=关键词%2Caps ← 搜索前缀 &sr=1-1 ← 排名位置 &tag=推广标签 ← 联盟标签 &xpid=xyz ← 页面ID

🔧 核心功能详解

2️⃣ 链接解析功能

可解析的参数

参数 含义 示例
ref 来源页面 sr_1_1 = 搜索第1页第1位
qid 搜索时间戳 Unix 时间戳
sr 搜索排名 1-1 = 第1页第1位
crid 搜索会话ID 16位随机字符
dib 设备指纹 Base64 编码的 JSON

🔬 技术实现原理

dib 参数详解

dib = Device/Browser Identifier

{ "v": "1", // 版本 "ct": "search", // 点击类型 "t": 1234567890, // 时间戳 "rk": 123456 // 排名标识 }

编码方式

  • JSON → Base64 → URL 编码
  • 包含设备指纹和会话信息
  • 由亚马逊 JavaScript 动态生成
注意：dib 参数只能通过真实浏览器获取，requests 无法模拟

⚠️ 风险与合规

亚马逊能看到的数据

数据类型 说明 风险等级
访问来源 搜索关键词、引荐页面 🟡 中

⚠️ 风险与合规

真正触发风控的因素

🔴 高风险行为

  • 新账号大量下单
  • 秒拍秒付（不浏览直接购买）
  • 同一 IP 多账号操作
  • 使用机房 IP 或代理 IP
  • 虚拟信用卡支付
  • 收货地址雷同
  • 评价内容雷同

🟢 安全行为

  • 使用老账号（6个月以上）
  • 正常的购买历史
  • 真实的收货地址
  • 自然的浏览行为

💡 最佳实践建议

1. 链接生成建议

✅ 应该做的：
  • 使用真实的关键词
  • 生成合理的搜索排名
  • 包含完整的时间戳和会话ID
  • 使用自然的浏览路径
❌ 不应该做的：
  • 使用相同的关键词重复生成
  • 时间戳过于规律
  • 排名位置不合理

❓ 常见问题

Q: 生成的链接安全吗？

A: 链接参数本身安全，但行为更重要。模拟真实用户行为是关键。

Q: 会被亚马逊封号吗？

A: 链接不会导致封号，但异常行为会。遵循最佳实践可降低风险。

Q: 支持哪些亚马逊站点？

A: 支持美国、中国、日本、英国、德国、法国、意大利、西班牙等。

Q: 短链服务稳定吗？

A: 自建短链服务完全可控，建议使用自己的域名。
